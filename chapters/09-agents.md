# 09. 에이전트: 손과 발이 생긴 AI

**핵심 비유**: 손발 없던 뇌에 비서/인턴의 몸이 붙음

**도입부 훅**: "ChatGPT 는 글만 쓰지만, Claude Code 는 진짜로 당신 컴퓨터에서 파일을 열고 명령어를 실행해요. 같은 AI 인데 뭐가 다를까요?"

**다룰 핵심 개념**: 에이전트(agent), 도구 사용(tool use), 하네스(harness), 반복 루프, Claude Code / ChatGPT plugins

**깜짝 사실 후보**:
- LLM 자체는 텍스트만 출력할 수 있고, 실제 실행은 하네스가 한다는 것
- "에이전트가 실수해도 되돌릴 수 있게" 만드는 것이 가장 어려운 엔지니어링 과제

**참고 자산**:
- `../../llm-field-notes/README.md` Part I 1장 "뇌와 몸: 에이전트의 해부학" — "손발 없는 뇌" 비유와 하네스 다이어그램. 단, 용어를 더 풀어 써야 함.
- `../../llm-field-notes/README.md` 의 "주문서를 쓰는 뇌" 도구 호출 설명 — 더 천천히 풀어냄.

**다이어그램 후보**:
- 순수 LLM vs 에이전트 비교: 왼쪽엔 `[입력] → [LLM] → [텍스트]` / 오른쪽엔 LLM 을 감싼 하네스 박스 (도구·반복 루프·파일 접근) — `llm-field-notes` 의 박스 그림 차용해 더 단순화
- 도구 호출 시퀀스 (Mermaid 시퀀스): 사용자 → LLM → "파일 읽어줘" 라는 주문서 → 하네스가 실제 실행 → 결과를 LLM 에 돌려줌 → LLM 이 다음 단계 결정
